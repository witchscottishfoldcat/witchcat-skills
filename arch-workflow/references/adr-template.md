# ADR Template

```text
ADR-<N>
  Decision : <what is being decided>
  Context  : <why the decision is needed now>
  Options  : <credible alternatives>
  Chosen   : <selected option>
  Tradeoff : <what is gained and what is sacrificed>
  Rollback : <how to revert if production behavior disproves the choice>
```

Keep the ADR concrete. Avoid generic statements like "more scalable" without the boundary, load shape, or failure mode that matters.
